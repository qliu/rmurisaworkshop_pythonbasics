ArcObjects Object Model Diagram (OMD)

ArcGIS products are built upon thousands of components called ArcObjects. When you use an application, such as ArcMap, usually you are looking at or working with ArcObjects. Every command or tool you use for adding data, editing, or performing spatial analysis is running code that includes ArcObjects.

ArcObjects are grouped into over 65 assemblies based on the relationships between the objects. Each assembly consists of ArcObjects that can be diagrammed based on their class relationships.

These diagrams are referred to as Object Model Diagrams (OMDs). OMDs provide information, such as how to work with certain classes and how classes relate to each other.

Carto Object Model: http://resources.arcgis.com/en/help/arcobjects-net/pdf/CartoObjectModel.pdf

Geometry Object Model: http://resources.arcgis.com/en/help/arcobjects-net/pdf/GeometryObjectModel.pdf

You can access OMDs from their install location at \Diagrams.
