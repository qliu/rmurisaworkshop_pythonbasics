Variable and Value

Variables and Values in ArcGIS

Variable is a storage location and an associated symbolic name that contains some known or unknown quantity or information (a value).

Value is an expression of actual quantity or information you are working with. 

            
variable = "value"

Python Identifiers

A Python identifier is a name used to identify a variable, function, class, module or other object.

An identifier starts with a letter A to Z or a to z or an underscore (_) followed by zero or more letters, underscores and digits (0 to 9).

Python does not allow punctuation characters such as @, $ and % within identifiers.
Python is a case sensitive programming language. Thus, Population and population are two different identifiers in Python. 
            
>>> a = 1
>>> a = 2
>>> print a
2
            
          
            
>>> a = 1
>>> A = 2
>>> print a
1
>>> print A
2

Reserved Words

Reserved words may not be used as constant or variable or any other identifier names. All the Python keywords contain lowercase letters only. Example: and, not, or, for, if, try, import, ...

Numbers (Numeric Types)

Numbers in ArcGIS

There are four distinct numeric types: plain integers (int), long integers (long), floating point numbers (float), and complex numbers (complex).

  • int (signed integers): often called just integers or ints, are positive or negative whole numbers with no decimal point.
  • long (long integers ): or longs, are integers of unlimited size, written like integers and followed by an uppercase or lowercase L.
  • float (floating point real values) : or floats, represent real numbers and are written with a decimal point dividing the integer and fractional parts. Floats may also be in scientific notation, with E or e indicating the power of 10 (2.5e2 = 2.5 x 102 = 250).
  • complex (complex numbers) : are of the form a + bJ, where a and b are floats and J (or j) represents the square root of -1 (which is an imaginary number). a is the real part of the number, and b is the imaginary part. Complex numbers are not used much in Python programming.

Basic Operations

+, -, *, /, %(mod), **(power), int(x) (convert x to integer), long(x), float(x), complex(x)

Boolean

Boolean values are the two constant objects False and True. In numeric contexts, they behave like the integers 0 and 1, respectively.

The built-in function bool() can be used to convert any value to a Boolean.

Strings

Strings in ArcGIS

Strings are a collection of one or more characters. String literals are written in single or double quotes. Python treats single quotes the same as double quotes.

Concatenation

            
>>> first_name = "Sherlock"
>>> last_name = "Holmes"
>>> full_name = first_name + " " + last_name
>>> print full_name
Sherlock Holmes

You cannot concatenate string and number values in Python.

              
>>> park_name = "wash_park"
>>> park_zip = 80209
>>> shape_ext = ".shp"
>>> park_file = park_name + park_zip + shape_ext

Traceback (most recent call last):
  File "", line 1, in 
    park_file = park_name + park_zip + shape_ext
TypeError: cannot concatenate 'str' and 'int' objects
In order to successfully concatenate these variables, you need to convert the park_zip value to a string. str() function tells Python to use the string equivalence of the value inside the parentheses. 
              
>>> park_file = park_name + str(park_zip) + shape_ext
>>> print park_file
wash_park80209.shp

String Formatting

The string format operator % produces formatted strings. 

            
>>> print "%s has sent you a shapefile: %s" % (full_name,park_file)
Sherlock Holmes has sent you a shapefile: wash_park80209.shp

Basic Format Symbol:
%c - character
%s - string
%d - signed decimal integer
%f - floating point real number

Accessing Values in Strings

Each of the characters in a string has an index that can be used to return the character.
The index starts with 0 and increases from left to right. 

            
>>> s = "Hello World!"
>>> print s[0]
H
>>> print s[1:5]
ello
>>> print s[:6]
Hello 
>>> print s[-1]
!
>>> print s[6:]
World!

List

List in ArcGIS

A list stores a sequence of items inside brakets and seperates the items using commas.

A list, like a string, is a collection of one or more items. Each item has an index that starts with 0 and increases from left to right.
Lists help you automate a task through a series of items. You can execute the task on each item, or you can call specific items using the item's index. 

            
>>> l = [1,2,3]
>>> l[0] = 4
>>> l
[4, 2, 3]
>>> l.append(5)
>>> l
[4, 2, 3, 5]

Tuple

Tuple in ArcGIS

A tuple stores a sequence of items.
Tuples are immutable, meaning you can't change it.
Tuples are useful to ensure the integrity of the sequence.
Similar to a list, a tuple is indexed and these indexs can be used to reference an item in a tuple. 

            
>>> t = (1,2,3)
>>> t[0] = 4

Traceback (most recent call last):
  File "", line 1, in 
    t[0] = 4
TypeError: 'tuple' object does not support item assignment

Dictionary

Dictionary in ArcGIS

Dictionary is a very useful data type built into Python. Unlike sequences, which are indexed by a range of numbers, dictionaries are indexed by keys, which can be any immutable type: strings, numbers, tuples (contain only strings, numbers, or tuples). You can’t use lists as keys, since lists can be modified in place.

It is best to think of a dictionary as an unordered set of key: value pairs, with the requirement that the keys are unique (within one dictionary). Placing a comma-separated list of key:value pairs within the braces adds initial key:value pairs to the dictionary. 

              
>>> neighborhood = {"FID": 0,"Shape": "Polygon","Name": "Athmar Park","TotalBirth": 183}
>>> neighborhood["Shape"]
'Polygon'
>>> neighborhood["Name"]
'Athmar Park'
>>> neighborhood["TotalBirth"]
183
